# 2026-04-25

- Created `MEMORY.md` as a compact long-term context file for Aurum sessions, covering Daniel preferences, product positioning, technical workflow, recent work, and workspace hygiene.
- Investigated the bug where Aurum answered the first writing prompt but appeared not to react to a follow-up. Root cause found in fragile SSE parsing and final turn insertion: both `/api/reflect` and the client assumed each `data:` event arrived in one network chunk, and the conversation continuation only updated an existing Aurum turn instead of adding one if no token had created it yet. Added a shared buffered SSE parser and made the continuation insert the final Aurum turn when needed.
- Follow-up evidence showed an Aurum reflection could still appear truncated mid-sentence while being treated as complete. Added an upstream completion guard: `/api/reflect` now requires DeepSeek `[DONE]` or `finish_reason` plus non-empty text before sending `done`, otherwise it sends a stream error and skips persistence/reply-count updates. The client now removes partial Aurum turns when a stream error arrives.
- Optimized follow-up reply latency: conversation replies now skip the pattern-detection prepass and existing-pattern fetch, keeping the first deep reflection richer while making "continuer avec Aurum" start faster. Follow-up max tokens were reduced from 1000 to 520, with very short replies still capped at 220.
- Addressed beta report UX issues: email signup now persists the accepted terms state and repairs it after email verification if the Cloud Function initialized legal settings later, preventing a duplicate CGU modal. Saving a page now starts the first Aurum reflection automatically, removing the confusing extra "Reveal" click after save.
