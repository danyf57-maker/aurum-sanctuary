# Aurum — Positioning Execution Brief

## Purpose

This document is the reference brief for translating Aurum's product strategy into concrete product and copy changes.

It is not a generic copy refresh.
It is a product-positioning correction.

The goal is to ensure Aurum is understood as a distinct premium category:

- AI-guided reflection
- emotional clarity
- private thinking companion
- inner pattern discovery

## 1) Strategic positioning to preserve

Aurum must not read as a standard journaling app.

It must read as a premium private reflection system that helps users:

- write privately
- receive thoughtful AI-guided reflection
- understand themselves more clearly
- notice recurring inner patterns over time

### Core rule

If the user mentally compares Aurum to Day One, Journey, or Reflectly as a journaling app, Aurum risks feeling expensive.

If the user mentally compares Aurum to:

- AI-guided reflection
- emotional clarity tools
- private self-understanding systems
- intelligent inner-companion products

then premium pricing becomes far more defensible.

## 2) Pricing context to respect

Current visible market references place classic journaling apps below Aurum's premium level.

So the product cannot justify its price mainly through:

- unlimited entries
- storage
- history
- export

The product must justify its price through:

- guided reflection
- emotional insight
- recurring pattern discovery
- privacy by design
- premium depth over time

### Strategic consequence

Aurum must move out of the user's mental category of “journal app”.

That category framing is more important than the feature list.

## 3) Priority ICP order

The ICPs are not equal in priority.

Recommended order:

1. Knowledge workers / creators
2. Premium self-care users
3. Power-journalers

### Why

Knowledge workers and premium self-care users are more likely to respond quickly to:

- clarity
- emotional decompression
- guided insight
- visible AI value

Power-journalers may value depth, but they are also harder to migrate because they already have strong habits, archives, and workflows.

## 4) Product structure to preserve

Aurum's product architecture should remain explicit and simple:

- `Write` = action
- `Journal` = memory
- `Magazine` = reading / analysis

This structure should guide future UI and copy decisions.

### Implication

Do not rebuild a vague “overview-first” product.
The center of gravity should remain:

- write
- revisit memory
- discover meaning and patterns

## 5) Core problem to solve

Current perception risk:
Aurum can still be read as a beautiful private journal for calm writing or stress relief.

Target perception:
Aurum should be read as a private AI-guided reflection companion that helps users write, understand themselves, and discover recurring inner patterns over time.

### Main gap to close

Two dimensions are still under-expressed and must become more visible:

- AI-guided reflection
- inner pattern discovery

## 6) UX rule: do not overload pages

This is a hard rule.

The solution is not to add more blocks everywhere.
The solution is to make existing surfaces clearer and more legible.

### Product rule

Prefer:

- sharper headlines
- clearer subheadlines
- stronger labels
- more meaningful empty states
- better value framing

Avoid:

- long educational sections
- dense explanation panels
- stacked feature noise
- copy that makes the interface feel heavy

### Guiding principle

Clarity before quantity.
Soft, but precise.
Premium, but not verbose.

## 7) Pages and surfaces to prioritize

### Priority 1 — Homepage

Goal:
A first-time visitor should understand in seconds that Aurum is:

- private
- AI-guided
- useful for emotional clarity
- designed to reveal patterns over time

#### What must change

Review and refine:

- hero headline
- hero subheadline
- support bullets
- “what Aurum is” explanation blocks
- proof/value framing

#### Acceptance test

A new visitor should be able to describe Aurum as:

“A private AI-guided reflection tool that helps me write, gain clarity, and see patterns over time.”

### Priority 2 — Pricing

Goal:
Pricing must feel justified by transformation, not just utility.

#### What must change

Pricing should clearly communicate:

- AI-guided reflections
- emotional clarity
- pattern discovery over time
- privacy-by-design architecture
- premium intelligence, not just premium storage

#### Acceptance test

A visitor should understand why Aurum is not priced like a standard journal app.

### Priority 3 — Onboarding

Goal:
Set the right expectation before the first meaningful use.

#### What onboarding should communicate

Aurum is:

- safe
- private
- intelligent
- reflective
- premium

#### Acceptance test

By the end of onboarding, the user should expect:

- to write privately
- to receive thoughtful AI guidance
- to notice recurring themes over time
- to gain clarity, not just save entries

### Priority 4 — Journal and Magazine

Goal:
Make the product's intelligence visible inside the real experience.

#### Journal should reinforce

- personal continuity
- meaningful memory
- private reflection archive

#### Magazine should reinforce

- recurring themes
- emotional patterns
- guided interpretation
- self-understanding over time

#### Acceptance test

The user should clearly perceive at least one of the following:

- recurring emotional themes
- repeated triggers
- changes over time
- structured reflection guidance

## 10) Recommended work method

### Step 1 — Audit current copy and UI framing

Inventory copy and product framing on:

- homepage
- pricing
- onboarding
- write flow
- journal
- magazine
- settings only if relevant

For each surface, classify whether it supports:

- privacy
- emotional clarity
- AI guidance
- pattern discovery

### Step 2 — Identify gaps

For each surface, answer:

- What is over-expressed?
- What is under-expressed?
- What is ambiguous?
- What weakens premium perception?
- What adds noise without adding clarity?

### Step 3 — Rewrite and rebalance

Adjust copy, labels, and emphasis so the four pillars are legible without adding unnecessary UI weight.

### Step 4 — Validate coherence

Check whether the same product is being described consistently across:

- homepage
- pricing
- onboarding
- write
- journal
- magazine

### Step 5 — Validate with simple test prompts

Use questions such as:

- Would a new US visitor still think this is just a journaling app?
- Would the user understand what the AI actually helps with?
- Would the user see why the pricing is premium?
- Would the user expect to discover patterns over time?

## 11) Definition of done

The work is done only if all of the following are true:

- Homepage clearly communicates the four positioning pillars.
- Pricing supports premium value perception.
- Onboarding sets the right expectation.
- Journal and Magazine visibly demonstrate reflective intelligence.
- Privacy remains a strong differentiator.
- The product no longer reads primarily as a standard journaling app.
- The pages feel clearer, not heavier.
